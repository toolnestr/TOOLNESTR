# 02 — The 30 Micro Tools

All tools run **100% in the browser** (no server, no database). That means: free hosting,
instant speed, user privacy (data never leaves their device — a great selling point), and
top SEO scores.

## Legend

- **Demand** = how much search traffic exists (🟢 high / 🟡 medium / 🔵 niche).
- **Competition** = how hard to rank (🟢 low / 🟡 medium / 🔴 high — we use long-tail to win).
- **Effort** = how hard to build (🟢 easy / 🟡 medium / 🔴 harder).
- **Phase** = when we build it (see roadmap).

## Build priority logic

We launch with **easy-to-build + high-demand + lower-competition** tools first so Google
starts indexing us quickly, then layer in the rest.

- **Phase 1 (launch, 3 tools):** #1 Word Counter, #2 Case Converter, #22 JSON Formatter.
  All easy, all popular, one per audience — proves the whole pipeline end to end.
- **Phase 2 (+5 tools):** #3 Age, #4 Date Diff, #5 Discount, #13 Fancy Text, plus polish.
- **Phase 3 (+ tools + AdSense):** #6 GST, #7 Tip, #8 EMI, #14 IG Bio, #15 Hashtag,
  #23 Base64, #24 URL, #25 QR.
- **Phase 4 (finish all 30 + growth):** everything remaining + supporting articles.

## How Batch 2 changes the plan
- **Total catalog: ~73 tools.** We do NOT build all of these — we keep building in phases and
  let Search Console data tell us which categories win, then go deep there.
- **Best low-competition bets to prioritize:** pet age (#31), sleep (#32), love calculator (#61),
  final grade (#39), cron (#67), JWT (#68), text repeater (#54), roman numerals (#59) — these
  are easy to build, genuinely low competition, and several are viral/shareable.
- **Best high-traffic bets (worth fighting long-tail):** image compressor/resizer, JPG→PDF,
  merge PDF, sleep calculator, GPA, salary converter, calorie/BMI bundle.
- **Heavy hitters for later:** PDF + image tools use small libraries (pdf-lib, browser canvas);
  we'll add them once the core text/calculator tools are live and indexed.
