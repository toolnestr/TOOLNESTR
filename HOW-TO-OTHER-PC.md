# Working on This Website From Another PC

This project lives on GitHub. GitHub is the "master copy" in the cloud.
Each computer just keeps a **working copy**. To move between PCs you:

1. **Pull** the latest copy before you start editing
2. Edit / preview locally
3. **Push** your changes back up when you finish

> Golden rule: **`pull` before you start, `push` when you finish.**
> That keeps every PC (and the live site) identical.

**Repository:** https://github.com/toolnestr/TOOLNESTR.git
**GitHub account to use:** `toolnestr` (sign in as this account, not any other)

---

## 1. One-time setup on a NEW PC

Install these three things:

| Tool | Why | Where |
|------|-----|-------|
| **Git** | downloads/uploads the code | https://git-scm.com |
| **Node.js** (LTS) | runs Astro to preview/build | https://nodejs.org |
| **VS Code** | edits files | https://code.visualstudio.com |

Easiest path for a no-coder: also install **GitHub Desktop**
(https://desktop.github.com). It does login + download + upload with
buttons instead of typed commands.

Sign in to GitHub once so it's allowed to upload your changes.

---

## 2. Get the project onto the new PC (do this once per PC)

### Option A — GitHub Desktop (recommended)
File → **Clone repository** → choose `toolnestr/TOOLNESTR` → **Clone**.

### Option B — command line
```bash
git clone https://github.com/toolnestr/TOOLNESTR.git
cd TOOLNESTR
npm install      # rebuilds the node_modules folder (not stored on GitHub)
```

`npm install` is required: the large `node_modules` folder is **not**
uploaded to GitHub on purpose, so each new PC rebuilds it from
`package.json`. This is normal.

---

## 3. Everyday workflow (any PC, every session)

```bash
git pull            # 1. get the latest before editing
npm run dev         # 2. preview at http://localhost:4321 (live-reloads as you edit)
git add -A          # 3. stage all your changes
git commit -m "describe what you changed"
git push            # 4. upload so other PCs / live site get it
```

In **GitHub Desktop** the last three steps are just:
type a summary → **Commit to main** → **Push origin**.

Useful commands:
- `npm run dev` — local preview while editing
- `npm run build` — produce the final site into the `dist/` folder
- `npm run preview` — preview the built site

---

## 4. Important cautions

- **Don't edit on two PCs at once** without pushing/pulling in between,
  or you'll get conflicts. Always `push` when you stop on one machine,
  and `pull` when you start on the other.
- **`dist/` and `node_modules/` don't travel between PCs.** They are
  regenerated by `npm install` and `npm run build`. Seeing them
  "missing" on a fresh clone is expected — not a problem.
- If `git push` asks you to log in, that's the normal first-time
  GitHub authentication. Sign in once and it remembers you.

---

## 5. Does pushing publish the live site?

If the live site (Cloudflare Pages) is connected to this GitHub repo,
then **every `git push` automatically rebuilds and publishes** the live
website — no extra step. If it is not connected, you publish manually.
Confirm this once so you know which case you're in.
